Across the Python codebase, there is only one third-party runtime library: plotly. The rest of the imports are standard-library modules, plus your own project modules like graph, metrics, and data_loader.

plotly.graph_objects is used in visualization.py to build the interactive 2D network map. The program creates go.Scatter traces in build_edge_trace, build_node_trace, and build_recommended_edge_trace to draw rail edges, station markers, and the suggested new connection. It then combines those into a go.Figure in create_network_figure, uses update_layout(...) to control titles, axes, aspect ratio, and legend, and finally uses write_html(...) and show() so the visualization can be exported or opened interactively. This library matters because it provides built-in interactive plotting primitives, hover text, marker color scales, and HTML export without your team having to build a plotting engine from scratch.

The GUI uses the standard-library tkinter package, imported as tk, plus ttk and messagebox in gui.py. tk.Tk() creates the main desktop window, tk.StringVar stores reactive UI state, ttk.Frame and ttk.Entry structure the layout, tk.Listbox powers autocomplete suggestions, and event methods like .bind(...) and trace_add(...) let the interface respond to typing and selection. That library is what makes the program a desktop app instead of a command-line script.

The GTFS parsing layer relies on standard-library csv, pathlib, collections, re, and dataclasses in data_loader.py. csv.DictReader reads stops.txt, stop_times.txt, routes.txt, and trips.txt row by row into dictionaries, which is why functions like load_stops, load_stop_times, and load_trips can access fields by column name such as row["stop_id"] and row["stop_lat"]. Path(...).open(...) from pathlib is used throughout to open files in a platform-safe way. defaultdict(list) and defaultdict(set) are used to group stop-times and route IDs without repeated existence checks. re.compile(...) is used to prebuild regex patterns that normalize platform-specific stop names into station-level names. @dataclass defines structured record types like StopTimeRecord, RouteRecord, and TripRecord, giving you typed containers for GTFS rows instead of loose tuples or dictionaries.

The graph and metrics code also depends heavily on dataclasses, heapq, and math. In graph.py, @dataclass and field(default_factory=...) define the Station and Graph data types, which hold the adjacency dictionary, station metadata, and route sets. In metrics.py, heapq.heappop and heapq.heappush implement Dijkstra's priority queue efficiently inside dijkstra(...). The math module provides math.inf for unreachable distances, math.isclose(...) for tie handling in shortest paths, and trigonometric functions like radians, sin, cos, atan2, and sqrt for the haversine-style geographic distance estimate used in edge recommendations.

A few other standard-library modules support smaller but important tasks. argparse in main.py uses ArgumentParser and add_argument(...) so users can pass file names and a data directory from the command line. difflib in gui.py is used for approximate station-name matching, which helps users recover from typos in the search UI. webbrowser opens the generated 3D HTML map in the user's browser from the GUI. json in visualization_3d.py serializes Python dict and list[dict] GeoJSON-like structures into JavaScript literals via json.dumps(...), which is how station and edge data are passed into the generated 3D webpage.

Finally, there are two development/testing imports that are not part of normal runtime behavior: doctest and python_ta. They appear only in the if __name__ == "__main__" blocks of several files and are used for module self-tests and static style checking through doctest.testmod() and python_ta.check_all(...).

So, the complete library/module inventory used in the Python codebase is: plotly.graph_objects, tkinter, ttk, messagebox, argparse, pathlib, csv, re, collections.defaultdict, dataclasses, heapq, math, difflib, webbrowser, json, doctest, python_ta, and __future__.annotations.
